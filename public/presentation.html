專案定位

「扭蛋樂園」模擬線上扭蛋機體驗：每台機台有多款公仔（不同稀有度與權重），使用者以帳戶餘額付費抽獎，獲得的公仔進入背包，可選擇以市價 60% 賣回平台，或付運費寄送到府／超商。

核心流程

註冊／登入→ 錢包儲值→ 選機台抽獎→ 背包管理→ 售出或出貨

為何符合課程要求

  • 前端：多個 HTML 表單與互動 UI（無 React/Vue，純 HTML/CSS/JS）
  • 後端：Express 接收 JSON，參數化 SQL 寫入資料庫
  • 資料庫：SQLite 正規化設計、外鍵、交易（transaction）保證一致性
  • 可現場演示：非靜態截圖，口試可即時操作完整流程

功能模組（9 項）

身分驗證 錢包儲值 機台瀏覽 伺服器端抽獎 背包 60% 售回 出貨下單 物流追蹤 管理員後台

  • 抽獎：權重隨機在伺服器計算，客戶端無法篡改結果
  • 售回：user_inventory.status → sold，餘額 + 交易紀錄同一 transaction
  • 出貨：宅配 90 元／超商 60 元；超商超過 20 件運費加倍、拆兩包裹
  • 管理員：檢視全站餘額與背包、更新任意出貨單狀態

資料庫如何被運用

資料表用途典型操作
users帳號、餘額、管理員註冊 INSERT；抽獎/出貨 UPDATE balance
machines機台主檔首頁 SELECT 列表
toys公仔款式與權重JOIN 抽獎池、顯示稀有度
user_inventory使用者持有的公仔實例抽獎 INSERT；售回 UPDATE status
transactions金流流水帳deposit / pull / sell / shipping
shipments出貨單主檔建立訂單、追蹤 status
shipment_items出貨 M:N 關聯一單多件；inventory_id UNIQUE

完整性與效能考量

  • PRAGMA foreign_keys = ON 保證參照完整性
  • 抽獎、售回、出貨使用 db.transaction()（ACID）
  • 餘額存在 users，歷史在 transactions — 方便錢包頁一次查詢

ER 關係總覽（概念圖）

與上一頁相同關聯結構，省略欄位細節，便於說明關聯與基數。

關聯基數
使用者 — 背包1 : N
使用者 — 交易紀錄1 : N
使用者 — 出貨單1 : N
扭蛋機 — 公仔1 : N
公仔 — 背包1 : N
出貨單 — 出貨明細 — 背包M : N（經明細解開）
USERS 使用者 MACHINES 扭蛋機 TOYS 公仔 USER_INVENTORY 背包 TRANSACTIONS 交易 SHIPMENTS 出貨 SHIPMENT_ITEMS 明細

設計理由

  • toys vs user_inventory：款式 vs 抽到的那一筆實例
  • shipment_items：標準 M:N 解開；UNIQUE(inventory_id) 防重複出貨
  • transactions.toy_id 可 NULL：儲值不需綁公仔
  • is_admin：使用者屬性即可，不必另開角色表

口試必考 Q2 — 專案實際使用的 SQL

選一個會顯示資料的頁面，說明對應 SELECT 為何這樣寫（JOIN、WHERE、ORDER BY）。

範例 A：首頁機台列表（index.html）

對應 routes/machines.js — 顯示所有機台：

SELECT * FROM machines ORDER BY machine_id

範例 B：我的背包（inventory.html）

多表 JOIN + 左連出貨明細，判斷是否「出貨中」：

SELECT ui.inventory_id, t.toy_id, t.name, t.rarity, t.price,
       t.image_url, ui.obtained_at, ui.status,
       m.name AS machine_name, si.shipment_id
FROM user_inventory ui
JOIN toys     t  ON t.toy_id      = ui.toy_id
JOIN machines m  ON m.machine_id  = t.machine_id
LEFT JOIN shipment_items si ON si.inventory_id = ui.inventory_id
WHERE ui.user_id = ? AND ui.status = 'owned'
ORDER BY ui.obtained_at DESC

說明：只顯示 owned；LEFT JOIN 因未出貨者無 shipment_items 列；依取得時間倒序。

範例 C：錢包流水（wallet.html）

SELECT t.transaction_id, t.type, t.amount, t.created_at,
       toy.name AS toy_name, toy.rarity AS toy_rarity
FROM transactions t
LEFT JOIN toys toy ON toy.toy_id = t.toy_id
WHERE t.user_id = ?
ORDER BY t.created_at DESC
LIMIT 50
